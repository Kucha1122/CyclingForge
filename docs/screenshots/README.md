# Zrzuty ekranu do README

Aby ładnie zaprezentować aplikację na GitHubie, dodaj tutaj zrzuty ekranu w formacie **PNG** lub **JPG**.

## Wymagane pliki (używane w głównym README)

| Plik | Co zrobić |
|------|------------|
| `dashboard.png` | Zrzut strony **Dashboard** (`/dashboard`) – PMC, podsumowania, rekomendacja. |
| `workout-today.png` | Zrzut **Rekomendacja na dziś** (`/workout/today`) – karta treningu, readiness. |
| `analysis.png` | Zrzut **Analiza** (`/analysis`) – wykres PMC, statystyki CTL/ATL/TSB. |
| `workouts.png` | Zrzut **Biblioteka treningów** (`/workouts`) – lista, filtry. |
| `workout-designer.png` | Zrzut **Designer treningu** (`/workouts/create` lub `/workouts/:id/edit`) – kreator kroków, wykres, strefy. |

## Jak zrobić zrzuty

1. Uruchom aplikację (frontend + backend).
2. Zaloguj się i przejdź do danej strony.
3. Zrób zrzut ekranu (np. **Win + Shift + S** w Windows, **Cmd + Shift + 4** na Macu) albo z pełnego okna przeglądarki.
4. Zapisz plik w tym folderze z dokładną nazwą jak w tabeli (np. `dashboard.png`).

## Wskazówki

- **Rozmiar:** ok. 1200 px szerokości wystarczy; GitHub wyświetli obraz w czytelnej skali.
- **Motyw:** możesz zrobić zrzuty w trybie jasnym lub ciemnym – wybierz ten, w którym UI wygląda lepiej.
- **Zawartość:** upewnij się, że na zrzucie widać charakterystyczne elementy (wykresy, karty, nawigacja), bez wrażliwych danych jeśli repo jest publiczne.
- **Dodatkowe zdjęcia:** możesz dodać więcej plików (np. `activities.png`, `sleep.png`) i dopisać je w sekcji „Podgląd aplikacji” w głównym `README.md`.

Po dodaniu plików i pushu na GitHub zrzuty pojawią się automatycznie w README.
